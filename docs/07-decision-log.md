# Decision Log

| ID      | Decision                                                            | Rationale                                                                                                                                                           | Date     | Alternatives Considered                            |
| ------- | ------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- | -------------------------------------------------- |
| DEC-001 | Use Firebase Firestore over a custom REST API                       | Real-time updates via `onSnapshot` without WebSocket infrastructure. Reduces backend complexity for a single-admin app.                                             | Mar 2026 | Supabase, custom Express API                       |
| DEC-002 | Inactive flag instead of delete for stylists/services               | Preserves referential integrity on historical bookings. Deactivated stylist's past bookings remain queryable and display correctly.                                 | Mar 2026 | Hard delete, soft delete with deleted_at timestamp |
| DEC-003 | Tiered pricing stored as object on service, not separate collection | Simpler reads — one document fetch gets all tiers. Price resolved at booking time and stored flat on the booking.                                                   | Mar 2026 | Separate pricing collection, junction table        |
| DEC-004 | Price history as append-only array on service document              | Full audit trail without a separate collection. `arrayUnion` ensures atomicity. Historical bookings reconcilable against price active on their date.                | Mar 2026 | Separate price_history collection                  |
| DEC-005 | All business rules in `config.ts`                                   | Single source of truth. Owner can request rule changes (trading hours, notice period, cutoff) without touching scheduling logic.                                    | Mar 2026 | Hardcoded in scheduling functions                  |
| DEC-006 | Scheduling engine in minutes from midnight                          | Avoids AM/PM string parsing bugs in time comparisons. All slot logic operates on integers.                                                                          | Mar 2026 | String comparison, Date objects                    |
| DEC-007 | Allow bookings during rest periods if new service fits              | Directly modelled from how Steve manages his floor — a colour client sitting with product does not block him from starting a men's cut. Significant revenue impact. | Mar 2026 | Block stylist entirely during rest period          |
| DEC-008 | Readable booking IDs: `{date}_{stylist-name}_{timestamp}`           | Human-readable in Firebase console. Easier debugging. No collision risk due to timestamp suffix.                                                                    | Mar 2026 | Auto-generated Firestore IDs                       |
| DEC-009 | `onSnapshot` for bookings, `getDocs` for stylists/services          | Bookings change frequently (real-time needed). Stylists and services change rarely (one-time fetch sufficient, with manual refetch after mutations).                | Mar 2026 | `onSnapshot` for all collections                   |
| DEC-010 | Optimistic UI updates with rollback                                 | Admin dashboard feels instant. Status changes reflect immediately without waiting for Firestore round-trip. Rollback restores previous state if write fails.        | Mar 2026 | Wait for Firestore confirmation before updating UI |
| DEC-011 | ToastContext over prop drilling for notifications                   | Any component can trigger a toast without threading `addToast` through every page component. Cleaner component interfaces.                                          | Apr 2026 | Prop drilling through page components              |
| DEC-012 | Denormalised stylist name and price on booking document             | Booking record is self-contained for analytics and display. Eliminates join queries. Accurate even after stylist deactivation or price change.                      | Mar 2026 | Join to stylists/services at query time            |
